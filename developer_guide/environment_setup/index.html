    🛠️ 개발 환경 설정

    NautilusTrader 개발 및 확장을 위한 환경 설정 방법을 안내합니다.

    ✅ 필수 도구

    개발을 시작하기 전에 다음 도구들을 준비하는 것을 권장합니다.

    • IDE: Cython 구문 분석을 지원하는 PyCharm Professional 또는 Visual Studio Code (Cython 확장 기능 설치)를 추천합니다.
    • 가상 환경: uv를 사용하여 Python 가상 환경과 의존성을 관리합니다.
    • 자동 검사: pre-commit을 사용하여 커밋 시점에 코드 포맷팅과 린팅을 자동으로 수행합니다.
    • Rust: 핵심 로직이 Rust로 작성되어 있으므로, Rust 설치 가이드에 따라 Rust를 설치해야 합니다.

    💡 중요: NautilusTrader는 Linux, macOS, Windows 모두에서 컴파일 및 실행되어야 합니다. 개발 시 항상 이식성을 염두에 두세요.

    🚀 설정 절차

    1. 프로젝트 및 의존성 설치: 다음 명령어를 사용하여 개발 및 테스트에 필요한 모든 의존성을 설치합니다. bash uv sync --active --all-groups --all-extras 또는 Makefile을 사용할 수 있습니다. bash make install > 🏃‍♂️ 빠른 개발을 위한 팁: 디버그 모드로 컴파일하면 최적화된 빌드보다 훨씬 빠르게 컴파일할 수 있습니다. 자주 코드를 수정하고 테스트한다면 디버그 모드 설치를 사용하세요. > bash > make install-debug >

    2. Pre-commit 훅 설정: 커밋할 때마다 자동으로 코드 품질 검사를 실행하도록 설정합니다. bash pre-commit install Pull Request를 보내기 전에는 다음 명령어를 실행하여 CI(지속적 통합) 검사를 통과할 수 있도록 코드를 정리하세요. bash make format make pre-commit

    📦 빌드

    Rust(.rs) 또는 Cython(.pyx, .pxd) 파일을 변경한 후에는 다시 컴파일해야 합니다.

    make build

    또는 더 빠른 디버그 모드 빌드를 사용하세요:

    make build-debug

    🐳 개발용 서비스 (Docker)

    .docker 디렉토리의 docker-compose.yml 파일을 사용하면 개발에 필요한 서비스를 쉽게 실행할 수 있습니다.

    docker-compose up -d

    위 명령은 다음 서비스들을 실행합니다: - postgres: PostgreSQL 데이터베이스 - redis: Redis 서버 - pgadmin: 데이터베이스 관리를 위한 웹 UI

    ⚠️ 주의: 이 Docker 설정은 개발용입니다. 운영 환경에서는 더 안전한 설정을 사용해야 합니다.

    ⌨️ Nautilus CLI

    Nautilus는 데이터베이스 관리 등을 위한 CLI(명령줄 인터페이스) 도구를 제공합니다.

    • 설치: make install-cli
    • 사용 예시 (데이터베이스 초기화): nautilus database init

    자세한 사용법은 nautilus --help로 확인할 수 있습니다.
