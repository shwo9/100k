    🏗️ 아키텍처

    NautilusTrader의 아키텍처 설계 원칙과 구조를 소개합니다.

    🏛️ 설계 철학

    NautilusTrader는 다음과 같은 현대적인 소프트웨어 설계 기법을 기반으로 구축되었습니다.

    • 도메인 주도 설계 (DDD)
    • 이벤트 기반 아키텍처
    • 메시징 패턴 (Pub/Sub, Req/Rep)
    • 포트와 어댑터 (Hexagonal Architecture)

    이러한 설계는 신뢰성, 성능, 모듈성, 테스트 용이성과 같은 품질 속성을 달성하기 위해 채택되었습니다.

    🗺️ 시스템 아키텍처

    NautilusTrader는 트레이딩 시스템을 구성하기 위한 프레임워크인 동시에, 다양한 환경에서 작동하는 기본 시스템 구현체의 집합입니다.

    핵심 컴포넌트

    • NautilusKernel: 모든 시스템 컴포넌트를 초기화하고 관리하는 중앙 지휘 센터입니다.
    • MessageBus: 컴포넌트 간의 통신을 담당하는 메시징 백본입니다. 이벤트와 데이터를 전파(Pub/Sub)하거나 요청/응답(Req/Rep) 통신을 처리합니다.
    • Cache: 상품, 계좌, 주문, 포지션 등 주요 상태 정보를 저장하는 고성능 인메모리 스토리지입니다.
    • DataEngine: 시장 데이터를 처리하고 필요한 컴포넌트에 전달하는 역할을 합니다.
    • ExecutionEngine: 주문의 생명주기를 관리하고, 리스크 검증 후 거래소로 주문을 라우팅합니다.
    • RiskEngine: 거래 전 리스크 검증, 포지션 및 노출도 모니터링 등 포괄적인 리스크 관리를 제공합니다.

    환경 컨텍스트 (Environment Contexts)

    NautilusTrader는 세 가지 실행 환경을 제공합니다.

    • Backtest: 과거 데이터를 사용하여 가상의 거래소에서 전략을 테스트합니다.
    • Sandbox: 실시간 데이터를 사용하지만 가상의 거래소에서 체결하여 전략을 테스트합니다.
    • Live: 실시간 데이터와 실제 거래소를 사용하여 라이브 트레이딩을 실행합니다.

    ⚙️ 코드 구조 및 흐름

    코드 구조

    NautilusTrader의 코드베이스는 성능과 안정성을 위해 두 개의 주요 언어로 구성됩니다.

    • Rust (crates 디렉토리): 핵심 로직, 계산, 저수준 처리를 담당합니다. Rust 컴파일러가 보장하는 메모리 및 타입 안전성 덕분에 매우 안정적입니다.
    • Python/Cython (nautilus_trader 디렉토리): Rust 코어를 C API를 통해 호출하여 사용합니다. 이를 통해 사용자는 친숙한 Python 환경에서 Rust의 성능을 그대로 활용할 수 있습니다.
    ┌───────────────────┐
│  nautilus_trader  │  <-- 사용자 영역 (Python/Cython)
│ (Python / Cython) │
└─────────┬─────────┘
          │ (C API 호출)
          ▼
┌───────────────────┐
│   nautilus_core   │  <-- 핵심 엔진 (Rust)
│      (Rust)       │
└───────────────────┘

    데이터 및 실행 흐름

    • 데이터 흐름: 외부 거래소(DataClient) → DataEngine 처리 → Cache 저장 → MessageBus 전파 → 전략/액터 소비
    • 실행 흐름: 전략의 주문 생성 → MessageBus 전송 → RiskEngine 검증 → ExecutionEngine 라우팅 → 외부 거래소(ExecutionClient) 제출

    프로세스와 스레드

    💡 팁: 최적의 성능을 위해 각 트레이더 인스턴스(하나의 NautilusKernel)는 별도의 프로세스에서 실행하는 것을 강력히 권장합니다. 시스템은 단일 스레드에서 매우 효율적으로 작동하도록 설계되었습니다.
