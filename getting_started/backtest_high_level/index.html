    ⬆️ 백테스트 (고수준 API)

    이 가이드는 BacktestNode를 사용하는 고수준(high-level) API를 통해 백테스트를 실행하는 방법을 안내합니다. 고수준 API는 운영 환경에 권장되며, 백테스트 설정을 라이브 트레이딩으로 쉽게 전환할 수 있도록 설계되었습니다.

    전체 코드는 backtest_high_level.ipynb Jupyter Notebook에서 확인하실 수 있습니다.

    📝 개요

    이 튜토리얼에서는 간단한 EMA 교차 전략을 시뮬레이션된 FX ECN 거래소에서 백테스트합니다. 고수준 API의 핵심은 데이터 카탈로그와 설정 객체를 사용하는 것입니다.

    • 외부 원시 데이터를 데이터 카탈로그에 로드하는 방법
    • BacktestNode를 위한 설정 객체들을 구성하는 방법
    • BacktestNode로 백테스트를 실행하는 방법

    🚀 고수준 API 백테스트 단계

    1. 데이터 로드 및 카탈로그 생성 📚

    고수준 API의 첫 단계는 데이터를 데이터 카탈로그에 저장하는 것입니다. 카탈로그는 데이터를 효율적으로 관리하고 재사용할 수 있게 해줍니다.

    1. 원시 데이터 로드: 외부 데이터(예: histdata.com의 FX 틱 데이터 CSV)를 pandas.DataFrame으로 읽어옵니다.
    2. 데이터 변환: QuoteTickDataWrangler를 사용하여 DataFrame을 Nautilus가 사용하는 QuoteTick 객체로 변환합니다.
    3. 카탈로그에 저장: ParquetDataCatalog 인스턴스를 생성하고, 변환된 상품 정보와 QuoteTick 데이터를 카탈로그에 씁니다.
    # 카탈로그 인스턴스 생성
catalog = ParquetDataCatalog(CATALOG_PATH)

# 상품 정보와 틱 데이터를 카탈로그에 저장
catalog.write_data([EURUSD])
catalog.write_data(ticks)

    2. 백테스트 환경 설정 객체 생성 ⚙️

    백테스트의 각 구성 요소를 별도의 설정 객체로 정의합니다. 이는 코드의 재사용성과 가독성을 높여줍니다.

    • BacktestVenueConfig: 시뮬레이션할 거래소(SIM)의 이름, 계좌 유형, 초기 자본 등을 설정합니다.
    • BacktestDataConfig: 데이터 카탈로그의 경로, 사용할 데이터(QuoteTick), 상품 ID, 백테스트 기간 등을 설정합니다.
    • ImportableStrategyConfig: 실행할 전략(EMACross)의 경로와 bar_type, trade_size 같은 파라미터를 설정합니다.

    3. 백테스트 실행 구성 묶기 📦

    BacktestRunConfig 객체를 사용하여 위에서 만든 모든 설정(엔진, 데이터, 거래소)을 하나로 묶어 백테스트 실행을 위한 완전한 구성을 만듭니다.

    config = BacktestRunConfig(
    engine=BacktestEngineConfig(strategies=strategies),
    data=data_configs,
    venues=venue_configs,
)

    4. 백테스트 실행 ▶️

    BacktestNode는 하나 이상의 BacktestRunConfig를 받아 백테스트 실행을 조정합니다. 이를 통해 여러 파라미터 조합을 한 번에 테스트하는 등 복잡한 작업을 쉽게 처리할 수 있습니다.

    # 설정 객체를 리스트에 담아 BacktestNode를 생성
node = BacktestNode(configs=[config])

# 백테스트 실행
results = node.run()

    5. 결과 확인 📈

    node.run() 메서드는 백테스트 결과를 담은 BacktestResult 객체의 리스트를 반환합니다. 이 객체를 통해 각 실행의 성과를 확인할 수 있습니다.
